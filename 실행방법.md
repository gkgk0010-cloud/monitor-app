# 실행 방법

## 1. 설치
```bash
cd monitor-app
npm install
```

## 2. 실행
```bash
npm run dev
```

브라우저에서 **http://localhost:3000/teacher/monitor** 로 접속.

## 3. Supabase Realtime
Supabase 대시보드 → Database → Replication → `student_status` 테이블이 publication에 포함되어 있어야 실시간 반영됨.

## 4. 테이블 구조 (참고)
`student_status` 테이블: `id`(PK), **`student_id`(text, unique)** ← 똑패스가 보내는 uid, upsert 기준, `student_name`, `student_color`, `info_text`, `last_active`(timestamptz).  
테이블에 `student_id` 컬럼이 없으면 추가 후 unique 제약 걸어야 똑패스 앱에서 상태가 정상 반영됨.

## 5. 똑패스 앱(GAS)에서 모니터 연동
학생이 퀴즈 풀 때 상태가 Supabase로 전송되려면, GAS **Script Properties**에 다음을 설정:
- `SUPABASE_URL`: Supabase Project URL (예: https://xxx.supabase.co)
- `SUPABASE_ANON_KEY`: Supabase Anon (public) key

설정 후 똑패스 웹앱에서 로그인하면 자동으로 모니터용 Supabase 클라이언트가 초기화됨.

## 6. 연결 흐름 (자세한 설명)
**똑패스_연결_가이드.md** 참고.  
똑패스(in-app) → Supabase `student_status` upsert ↔ 선생님 모니터가 같은 테이블 조회 → 명단·공지 복사 등.
