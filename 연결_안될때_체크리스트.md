# 똑패스 ↔ Supabase 연결이 안 될 때

## 1. 테이블에 컬럼이 다 있는지

똑패스 앱은 아래 **5개 필드**를 보냅니다. 테이블에 없으면 요청이 실패합니다.

| 컬럼 | 타입 | 필수 |
|------|------|------|
| `student_id` | text (unique) | ✅ |
| `student_name` | text | ✅ |
| `student_color` | text | ✅ |
| `last_active` | timestamptz 또는 timestamp | ✅ |
| `info_text` | text | ✅ |

**지금 화면에 `last_active`, `info_text` 가 안 보이면** → 아래 SQL을 **SQL Editor**에서 실행하세요.

```sql
ALTER TABLE public.student_status
ADD COLUMN IF NOT EXISTS last_active timestamptz;

ALTER TABLE public.student_status
ADD COLUMN IF NOT EXISTS info_text text;
```

---

## 2. GAS Script Properties

- **똑패스**가 배포된 GAS 프로젝트 → **프로젝트 설정** → **스크립트 속성**
- 다음 두 개가 **정확히** 들어가 있어야 합니다.
  - `SUPABASE_URL` = Supabase Project URL (예: https://xxxxx.supabase.co)
  - `SUPABASE_ANON_KEY` = Supabase anon public key
- 하나라도 비어 있거나 오타면 Supabase에 연결되지 않습니다.

---

## 3. 똑패스에서 로그인했을 때

- **브라우저 개발자 도구** (F12) → **Console** 탭
- 로그인 직후 `모니터 push 실패:` 같은 **빨간 에러**가 있으면:
  - 메시지에 "column ... does not exist" → 위 1번처럼 **컬럼 추가**
  - "Invalid API key" / "JWT" → GAS Script Properties **SUPABASE_ANON_KEY** 다시 복사
  - "Failed to fetch" / "CORS" → Supabase URL이 **https**인지, 오타 없는지 확인

---

## 4. 한 번에 확인하는 순서

1. Supabase **Table Editor** → `student_status` 에 **student_id, student_name, student_color, last_active, info_text** 다 있는지 확인  
   → 없으면 위 SQL로 `last_active`, `info_text` 추가
2. GAS **스크립트 속성**에 **SUPABASE_URL**, **SUPABASE_ANON_KEY** 넣었는지 확인
3. 똑패스 웹앱에서 **로그인** → F12 Console 에러 확인
4. Supabase **Table Editor**에서 `student_status` **새로고침** → 새 행 생겼는지 확인

---

## 5. 똑패스 로그인 후 F12로 어디서 막히는지 보기

똑패스 **웹앱** 탭에서 **로그인** 한 직후, **F12 → Console** 에서 아래 메시지가 **순서대로** 나오는지 확인하세요.

1. `모니터: 초기화 시도 (로그인 직후)` → init 실행됨
2. `모니터: config 수신 URL있음 KEY있음` → GAS에서 설정 받음
3. `모니터: Supabase 연결됨` → 클라이언트 생성됨
4. `모니터: push 완료 [이름]` → Supabase에 저장됨

**여기서 막히는 곳:**
- **2번에서 "URL없음 KEY없음"** → GAS Script Properties 확인, 또는 **Code.js 저장 후** 똑패스 다시 열기
- **"모니터: getMonitorConfig 실패 (GAS)"** → GAS에 `getMonitorConfig()` 함수가 있는지, **배포한 앱**으로 접속했는지 확인
- **"push 스킵 (Supabase 미연결)"** → 2~3번이 실패한 상태. config 또는 supabase CDN 로드 확인
- **"모니터 push 실패:"** 뒤에 에러 메시지 → Supabase 테이블 컬럼/권한 또는 점검 시간대 확인
