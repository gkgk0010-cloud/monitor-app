# 모니터: 집중 관리 존 vs 안전 보관함 기준

## 1. 기준 요약

| 구분 | 기준 | 표시 |
|------|------|------|
| **집중 관리 존 (주요관리 30인)** | **최근 활동 순** 상위 30명 (`last_answer_at` 또는 `last_active` 기준, **가장 최근에 문제 푼 사람이 1번**) | 카드 30장 · 풀면 위로 올라옴, 31번째는 안전으로 내려감 |
| **안전 보관함** | 그 외 전원 (31번째 이후) | 접기/펼치기 목록 |

- **상위 30명 기준**: **방금 전에 활동(문제 풀이)한 시각**이 가장 최근인 사람이 1번, 그다음이 2번 … 30번까지. **문제를 풀면** `last_answer_at` / `last_active`가 갱신되어 **맨 위로 올라옴**. 안전 보관함에 있던 사람도 문제 풀기 시작하면 위로 올라와서 집중관리존에 보이고, 기존 30번은 31번이 되어 안전 보관함으로 내려감.
- **같은 모습만 나오는 문제 해결**: 이전에는 "색깔 순" 또는 "오늘 접속"만 쓰면 같은 30명이 고정돼 있었음. **최근 활동 순**으로 바꿔서, **지금 막 푼 사람**이 항상 위에 오고, 순차적으로 내려가게 함.

## 2. 정렬 기준 (집중 관리 존)

- **1~30번**: `last_answer_at`(문제 푼 시각) 우선, 없으면 `last_active`(마지막 활동 시각) 기준 **내림차순** (가장 최근이 1번).
- 활동 시각이 없거나 오래된 사람은 뒤로 밀려나서 안전 보관함에 표시됨.

## 3. 이틀 미접속

- **기준**: `last_active`가 **오늘 기준 2일 전 이전**이면 "이틀 미접속" (한국 시간 기준).
- **표시**: 집중 관리 존·안전 보관함 카드에 "이틀 미접속" 뱃지 + 별도 섹션 "이틀 연속 미접속" 목록.
- **멘트 복사**: "이틀 미접속 멘트 복사" 버튼 → 이틀 미접속 학생 이름 + 보낼 멘트(개별 발송용)가 클립보드에 복사됩니다.

## 4. 이미 undefined 되어 있는 학생 (시트)

- 시트에 "undefined"가 들어간 학생은 **GAS**에서 처리합니다.
- **찾기**: GAS 편집기에서 `findUndefinedInProfile` 실행 → 로그에 "시트명 / 행 / uid / 해당 열" 목록 출력.
- **한 번에 고치기**: GAS에서 `cleanUndefinedInProfile` 실행 → User_Profile 시트들에서 "undefined" 문자열이 빈 칸으로 치환됩니다.
