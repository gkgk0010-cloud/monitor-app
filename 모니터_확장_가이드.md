# 모니터 확장 가이드 (실시간 색깔 깜빡임 · 오늘 스코어 · 태그 약점)

현재 구현된 것과, "바삐 색깔 깜빡임"·"오늘 스코어"·"오늘 약점 Worst 3"을 쓰려면 필요한 DB/똑패스 앱 연동을 정리했습니다.

---

## 📌 연동 방법 (요약)

**오늘의 스코어 · 오늘의 약점**을 상세 팝업에 띄우려면:

1. **Supabase**  
   - `daily_stats` 또는 `answer_logs` 테이블 생성 (가이드 아래 "오늘의 스코어 · 오늘의 약점" 참고).  
   - `student_id`, 날짜(한국), 문제 수/정답 수 또는 태그별 오답 수를 저장.
2. **똑패스 앱(GAS)**  
   - 문제 제출 시 위 테이블에 insert/upsert (스프레드시트에 이미 있는 태그·정오답 정보를 Supabase로 전송).
3. **모니터 앱**  
   - 상세 팝업 열릴 때 해당 학생 + 오늘 날짜(한국)로 Supabase 조회 후, "오늘의 스코어"·"오늘의 약점" 문구를 실제 숫자/태그로 채우면 됩니다.

**집중관리존 빨간불/파란불**은 `student_status`에 `last_answer_result`, `last_answer_at` 추가 후, GAS에서 문제마다(또는 쓰로틀) 전송하고 모니터에서 애니메이션만 붙이면 됩니다.

---

## ✅ 이미 구현된 것

- **실시간 사건 기록**  
  - 각 행에 **날짜+시간** 표시 (어제/오늘 구분 가능).  
  - 각 행에 **💬 복사** 버튼 → 그 로그에 맞는 카톡 개별멘트(복습완료/새벽반/연속오답 등) 복사, **학생명 자동 반영**.
- **오늘 출석·미접속 복사** → 상단 헤더에만 유지 (아래 별도 섹션 제거됨).
- **학생 상세 팝업** → 집중관리존 **카드 클릭** 시 "🕵️‍♂️ OOO 상세" 모달
  - **📜 개인 로그**: 해당 학생의 `status_logs`만 필터, **날짜+시간** 표시.
  - **📊 오늘의 스코어** · **📉 오늘의 약점 (Worst 3)**: UI만 있고, 문구는 "데이터 수집 연동 예정" (위 연동 후 채움).
- **카드 클릭** → 상세 팝업 / **💬 복사** 버튼만 누르면 복사만 되고 팝업은 안 열림.

## 🔜 집중관리존 "바삐 색깔 깜빡임" (문제 풀이마다 빨/초 등)

**원하는 동작:** 문제 풀이할 때마다 카드가 잠깐 **정답=초록, 오답=빨강** 등으로 깜빡이다가, 기존 상태(복습완료·새벽반 등) 색으로 돌아오기.

**필요한 작업:**

1. **Supabase `student_status` 컬럼 추가**
   - `last_answer_result` (boolean, nullable) — 마지막 정답 여부
   - `last_answer_at` (timestamptz, nullable) — 마지막 풀이 시각
2. **똑패스 앱(GAS)**  
   - 매 문제 제출 후 `pushMonitorStatus` 할 때 위 두 값도 함께 보내기 (기존 색/멘트 유지하면서 `last_answer_result`, `last_answer_at`만 갱신).
   - **속도/부하:** 문제마다 1회 업데이트하면 됨. 학생 수 × 풀이 수만큼 요청이 가므로, 필요하면 **학생당 2~3초에 1번**처럼 쓰로틀 걸어도 됨.
3. **모니터 앱**  
   - `student_status` 구독 시 `last_answer_result` / `last_answer_at` 사용
   - 해당 학생 카드에 `last_answer_at`이 최근 2~3초 이내면 `last_answer_result`에 따라 짧게 초록/빨강 테두리(또는 배경) 애니메이션 후, 기존 `student_color` 스타일로 복귀

이렇게 하면 "느릴까?"는 **쓰로틀만 적당히 주면** 무리 없이 가능한 수준입니다.

## 🔜 오늘의 스코어 · 오늘의 약점 (Worst 3)

**원하는 동작:** 상세 팝업에서  
- "오늘 N문제 풀었고, 정답률 80% (28정답 / 7오답)"  
- "오늘 유독 **가정법**에서 많이 틀렸어요 (가정법 3개, 관계사 1개 …)"

**필요한 작업:**

1. **Supabase 테이블 (둘 중 하나 또는 조합)**
   - **방법 A)** `daily_stats`  
     - `student_id`, `date`(한국기준 날짜), `problems_solved`, `correct_count`, `wrong_count`  
     - 똑패스 앱에서 문제 제출 시 해당 날짜 행을 upsert
   - **방법 B)** `answer_logs`  
     - `student_id`, `created_at`, `tag`(예: 가정법), `correct`(boolean)  
     - 문제마다 1행 insert (나중에 오늘 스코어·태그별 오답 수 집계)
2. **똑패스 앱(GAS)**  
   - 스프레드시트/기존 DB에 **태그**가 있으니, 제출 시 위 테이블에 기록 (일별 집계만 할지, 문제별 로그만 할지 정하면 됨).
3. **모니터 앱**  
   - 상세 팝업 열릴 때 `student_id`(또는 학생 식별자) + 오늘 날짜(한국시간)로 `daily_stats` / `answer_logs` 조회
   - **오늘의 스코어**: `problems_solved`, `correct_count`, `wrong_count`, 정답률 계산해서 문구로 표시
   - **오늘의 약점 Worst 3**: `answer_logs`에서 오늘·해당 학생·오답만 모아 태그별 개수 세고 상위 3개 표시

DB에 태그가 있으니, 위 테이블만 설계해서 GAS에서 채워 주면 "어떤 태그 틀렸는지"까지 그대로 보여줄 수 있습니다.
