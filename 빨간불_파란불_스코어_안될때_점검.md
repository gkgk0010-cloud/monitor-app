# 빨간불/파란불·오늘의 스코어·실시간 사건 기록이 안 될 때 점검

**증상:** 개인 로그(새벽 공부 시작 등)는 보이는데, 집중관리존 빨간불/파란불이 안 바뀌고, 상세 팝업 "오늘의 스코어"가 0문제로 나옵니다.  
**또는:** "실시간 사건 기록"이 "아직 기록된 사건이 없어요"만 나오고, 잘 되다가 갑자기 안 뜹니다.

---

## ⭐ "오늘의 스코어"만 0 / 얼마나 풀었는지 연결 안 될 때 (빠른 점검)

| 순서 | 확인 | 할 일 |
|------|------|--------|
| 1 | **answer_logs 테이블** | Supabase Table Editor → **answer_logs** 테이블이 있는지 확인. 없으면 **2-2** 아래대로 `supabase_answer_logs.sql` 실행. |
| 2 | **quiz_type 컬럼** | answer_logs에 **quiz_type** 컬럼이 있는지 확인. 없으면 `monitor-app/supabase_answer_logs_quiz_type.sql` 실행. |
| 3 | **GAS Script Properties** | Apps Script → 프로젝트 설정 → Script properties에 **SUPABASE_URL**, **SUPABASE_ANON_KEY** 가 들어가 있어야 똑패스 앱이 모니터용 Supabase에 쓸 수 있음. (개인 로그는 되는데 스코어만 안 되면 이건 보통 맞음.) |
| 4 | **실제로 insert 되는지** | 학생이 **문제 1~2개 풀고** → Supabase Table Editor에서 **answer_logs** 테이블 열어서 **해당 학생(student_id)으로 새 행이 쌓였는지** 확인. 안 쌓이면 학생 앱 F12 → Console에서 `모니터 answer_log 실패:` 메시지 확인. |
| 5 | **같은 Supabase 프로젝트** | 똑패스 앱(GAS Script Properties)과 모니터 앱(.env.local의 NEXT_PUBLIC_SUPABASE_URL)이 **동일한 Supabase 프로젝트**를 바라봐야 함. 다르면 똑패스는 A에 쓰고 모니터는 B를 보게 되어 0으로 보임. → 아래 **3-4** 참고. |

## 0. 빨간불/파란불 기능 점검 (요약)

**흐름:** 학생이 문제 풀기 → 똑패스 앱 `pushAnswerResultToMonitor(isCorrect, tag)` 호출 → Supabase **student_status** upsert (`last_answer_result`, `last_answer_at`, `last_answer_tag`) → 모니터 앱이 **student_status** 구독/조회 → **최근 20초 이내** 답안이 있으면 정답=파란불, 오답=빨간불, 그 외=대기(회색).

**체크리스트 (빨간불/파란불이 안 보일 때):**

| 순서 | 확인 항목 | 할 일 |
|------|-----------|--------|
| 1 | Supabase **student_status** 컬럼 | Table Editor에서 **last_answer_result**, **last_answer_at**, **last_answer_tag** 가 있는지 확인. 없으면 **2-1** 실행 (`supabase_student_status_answer_columns.sql`). |
| 2 | 똑패스 앱 콘솔 | 학생 앱에서 문제 1개 풀고 F12 → Console. `모니터 answer 결과 push 실패:` 가 **안** 나와야 함. 나오면 Supabase(컬럼/RLS) 점검. |
| 3 | 모니터 Realtime | **student_status** 가 **Database → Publications → supabase_realtime** 에 포함돼 있어야 새로고침 없이 불이 바뀜. (이미 켜 두었으면 생략.) |
| 4 | 불 표시 시간 | 파란불/빨간불은 **마지막 답안 제출 후 20초**까지만 표시되고, 이후 자동으로 "대기"로 바뀝니다. 더 오래 보이게 하려면 모니터 앱의 `ANSWER_LIGHT_SECONDS` 값을 키우면 됩니다. |

## 1. 원인 정리

| 기능 | 필요한 것 | 없을 때 |
|------|-----------|---------|
| **빨간불/파란불** | `student_status` 테이블에 **last_answer_result, last_answer_at, last_answer_tag** 컬럼 | upsert 시 해당 컬럼이 없어서 **실패** → 불 안 바뀜 |
| **오늘의 스코어·약점** | **answer_logs** 테이블 존재 + 문제 풀 때마다 insert | 테이블 없거나 insert 실패 → 0문제 |

개인 로그·실시간 사건 기록은 **status_logs** 테이블입니다.  
**실시간 기록이 갑자기 안 뜨는 경우:** 앱이 **student_id**를 보내는데, Supabase **status_logs** 테이블에 **student_id 컬럼이 없으면** INSERT가 실패해서 새 행이 안 쌓입니다.  
→ 똑패스 앱은 지금 **student_id를 안 보내도록** 되어 있어서, 컬럼 없는 테이블에서도 INSERT가 됩니다. (컬럼 추가 후 student_id를 다시 보내려면 gas_src index.html의 pushStatusLog에 student_id를 넣으면 됩니다.)

## 2. 반드시 할 일 (Supabase)

### 2-1. 빨간불/파란불용 컬럼 추가 (한 번만)

1. Supabase 대시보드 → **SQL Editor**
2. **`monitor-app/supabase_student_status_answer_columns.sql`** 파일 내용 **전체 복사** → 붙여넣기 → **Run**
3. **Success** 나오면 `student_status` 에 **last_answer_result**, **last_answer_at**, **last_answer_tag** 컬럼이 생깁니다.

→ 이걸 **안 하면** 학생 앱이 문제 풀 때마다 보내는 upsert가 **실패**하고, 집중관리존은 계속 "대기"만 나옵니다.

### 2-2. 오늘의 스코어용 테이블 생성 (한 번만)

1. Supabase 대시보드 → **SQL Editor**
2. **`monitor-app/supabase_answer_logs.sql`** 파일 내용 **전체 복사** → 붙여넣기 → **Run**
3. **Success** 나오면 **answer_logs** 테이블이 생깁니다.

→ 이걸 **안 하면** "오늘의 스코어"는 항상 0문제로 나옵니다.

### 2-3. 실시간 사건 기록용 테이블 (info_text는 되는데 실시간 기록만 안 뜰 때)

**증상:** 집중관리존 카드의 **info_text**(접속 중, 🏆 일일 할당량 클리어! 등)는 잘 바뀌는데, **실시간 사건 기록**은 "아직 기록된 사건이 없어요"만 나옵니다.

- **info_text**는 **student_status** 테이블(upsert)에 저장됩니다.
- **실시간 사건 기록**은 **status_logs** 테이블(insert)에 저장됩니다. **별도 테이블**이라, status_logs가 없으면 INSERT가 실패하고 기록이 안 쌓입니다.

**할 일:**

1. Supabase 대시보드 → **Table Editor** → **status_logs** 테이블이 있는지 확인.
2. **없으면** → **SQL Editor**에서 **`monitor-app/supabase_status_logs.sql`** 내용 전체 복사 → 붙여넣기 → **Run**.
3. **Success** 나오면 **status_logs** 테이블이 생기고, 이후부터 실시간 사건 기록이 쌓입니다.
4. 학생 앱에서 한 번 로그인·문제 풀기·샷건 종료 등 하면, 브라우저 **F12 → Console**에서 `모니터 status_logs 실패: 404 ...` 같은 메시지가 **사라지는지** 확인.

**갱신 버튼을 눌러도 실시간 사건 기록이 안 뜨는 경우**

- 모니터 앱은 조회 실패 시 **실시간 사건 기록** 섹션에 빨간 박스로 에러 메시지를 띄웁니다. (예: `permission denied for table status_logs`, `relation "status_logs" does not exist` 등.)
- **할 일:** 1) 갱신 버튼 한 번 누른 뒤 뜨는 에러 문구 확인. 2) Supabase **Table Editor**에서 **status_logs** 테이블이 있는지 확인. 3) **status_logs** 테이블 선택 → **Policies** 탭에서 **anon**용 **SELECT** 정책이 있는지 확인. 없으면 **`monitor-app/supabase_status_logs.sql`** 전체를 SQL Editor에서 실행해 테이블·RLS 정책을 만드세요.

**status_logs에 데이터(message)는 잘 들어가는데 모니터에만 안 뜨는 경우**

- **모니터 앱**은 `.env.local`의 `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`로 조회합니다.
- **똑패스 앱**은 GAS `getMonitorConfig()`가 주는 URL/키로 INSERT합니다.
- 이 둘이 **서로 다른 Supabase 프로젝트**면, 똑패스는 A에 쓰고 모니터는 B를 보게 되어 **실시간 기록이 안 나옵니다.**
- **같은 프로젝트인지 확인하는 방법** (아래 "3-4. 같은 Supabase 프로젝트 쓰는지 확인" 참고).

### 2-4. message·infotext는 즉시 잘 바뀌는데, 실시간 사건 기록만 "연결 끊긴 것처럼" 즉시 안 뜨는 경우

**증상:** 집중관리존의 **message**(접속 중, 🏆 일일 할당량 클리어! 등), **info_text**는 **즉각적으로** 잘 바뀌는데, **실시간 사건 기록**은 새로고침하거나 갱신 버튼을 눌러야만 보입니다. (또는 "아직 기록된 사건이 없어요"만 나옵니다.)

**원인:** **student_status**는 Realtime Publication에 포함돼 있어서 즉시 반영되고, **status_logs**는 Publication에 없으면 INSERT는 되지만 모니터가 **INSERT 이벤트를 받지 못해** 실시간으로 안 뜹니다.

**할 일 (한 번만):**

1. Supabase 대시보드 → **SQL Editor**
2. **`monitor-app/supabase_realtime_status_logs.sql`** 내용 전체 복사 → 붙여넣기 → **Run**
3. 실행 내용: `ALTER PUBLICATION supabase_realtime ADD TABLE public.status_logs;`
4. **Success** 나오면 이후부터 실시간 사건 기록이 **새로고침 없이** 즉시 뜹니다.

(이미 `status_logs`가 Publication에 있으면 "already in publication" 같은 오류가 날 수 있습니다. 그 경우는 이미 설정된 것이므로 무시해도 됩니다.)

## 3. 확인 방법

### 3-1. Supabase Table Editor에서

- **student_status** 테이블 열기 → 컬럼 목록에 **last_answer_result**, **last_answer_at**, **last_answer_tag** 가 있는지 확인.
- **answer_logs** 테이블이 있는지, 학생이 문제 푼 뒤 **새 행이 쌓이는지** 확인.

### 3-2. 학생 앱에서 문제 한두 개 풀고

- 브라우저 **개발자 도구(F12)** → **Console** 탭
- 다음 메시지가 **안** 나오는지 확인:
  - `모니터 answer 결과 push 실패: ...`
  - `모니터 answer_log 실패: ...`
- 위 메시지가 **나오면** → Supabase 쪽 오류(컬럼 없음, 테이블 없음, RLS 등) 가능성이 큽니다.

### 3-3. 모니터 앱 (Realtime Publication)

- **student_status** 가 **Database → Publications → supabase_realtime** 에 포함돼 있어야 **새로고침 없이** 빨간불/파란불·message·info_text가 갱신됩니다.
- **status_logs** 도 **supabase_realtime** 에 포함돼 있어야 **실시간 사건 기록**이 새로고침 없이 즉시 뜹니다. (message·infotext는 되는데 실시간 기록만 안 뜨면 → `supabase_realtime_status_logs.sql` 실행.)

### 3-4. 같은 Supabase 프로젝트 쓰는지 확인 (똑패스 vs 모니터)

**똑패스 앱**은 GAS Script Properties의 `SUPABASE_URL` / `SUPABASE_ANON_KEY`로, **모니터 앱**은 `.env.local`(또는 Vercel 환경 변수)의 `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`로 DB에 접속합니다. 둘이 **다르면** 똑패스는 A 프로젝트에 쓰고 모니터는 B를 보게 되어 실시간 기록이 안 뜹니다.

**확인 절차:**

1. **GAS(똑패스) 쪽 URL**
   - Apps Script 에디터 → **프로젝트 설정**(톱니바퀴) → **Script properties** → `SUPABASE_URL` 값 확인.  
   - 또는 임시로 `Code.js`에 아래 넣고 실행 → **보기 → 로그**에서 URL 확인:
     ```js
     function _checkMonitorUrl() {
       var c = getMonitorConfig();
       console.log('GAS SUPABASE_URL:', c.supabaseUrl);
     }
     ```
2. **모니터 쪽 URL**
   - 로컬: `monitor-app/.env.local` 열어서 `NEXT_PUBLIC_SUPABASE_URL=` 뒤 값 확인.
   - Vercel 배포: Vercel 대시보드 → 해당 프로젝트 → **Settings → Environment Variables** → `NEXT_PUBLIC_SUPABASE_URL` 값 확인.
3. **비교**
   - 두 URL이 **완전히 동일**해야 합니다. (예: `https://xxxxx.supabase.co` 끝에 슬래시 유무까지 맞추기.)
   - 다르면 모니터 `.env.local`(또는 Vercel 변수)을 GAS의 `SUPABASE_URL`과 **같게** 수정 후, 로컬은 재시작, Vercel은 **Redeploy** 한 번 해주세요.

### 3-5. "뭘 건드렸길래 갑자기 이렇게 됐어?" (실시간 기록 관련)

**코드에서 바꾼 것:**  
똑패스 앱은 `status_logs` INSERT 시 `created_at`(서버 기본값 쓰도록 제거), `student_id`(테이블에 컬럼 없을 때 INSERT 실패 방지로 제거). 모니터 앱은 `status_logs` 조회 시 SELECT 컬럼만 정리. → **실시간 구독(Realtime) 자체를 끊거나 바꾼 수정은 없음.**

**"갑자기" 안 됐다면 보통 원인은:**

1. **Realtime Publication**  
   `status_logs`가 `supabase_realtime`에 없었던 경우. (SQL 실행 시 "already member" 나오면 이미 들어가 있는 거라 그건 아님.)
2. **폰/탭·브라우저 환경**  
   모바일·백그라운드 탭에서는 WebSocket이 끊겨서 실시간 이벤트를 못 받을 수 있음. → **갱신** 버튼으로 수동 반영.
3. **프로젝트 불일치**  
   URL/키는 저절로 안 바뀜. 다만 **처음 설정·배포할 때** GAS와 모니터에 서로 다른 Supabase 프로젝트를 넣어 둔 경우에만 해당. (평소에 잘 되다가 갑자기 안 됐다면 1·2가 더 가능성 있음.)

## 5. 데스크톱은 되는데 폰에서만 안 될 때

**증상:** PC에서는 빨간불/파란불이 실시간으로 잘 바뀌는데, 모니터를 **폰(모바일 브라우저)**로 보면 안 바뀌고 새로고침해야만 보임.

**이유:** 모바일 브라우저는 배터리 절약 등으로 **Realtime(WebSocket) 연결을 자주 끊거나**, 백그라운드 탭을 멈춥니다. 그래서 폰에서 모니터를 볼 때 실시간 이벤트를 못 받을 수 있습니다.

**대응 (모니터 앱 쪽):**

1. **탭 복귀 시 자동 갱신**  
   폰에서 다른 앱 갔다가 모니터 탭으로 돌아오면, 자동으로 `student_status`를 다시 불러와서 빨간불/파란불이 최신으로 갱신됩니다.

2. **🔄 갱신 버튼**  
   집중 관리 존 제목 옆 **「🔄 갱신」** 버튼을 누르면, 새로고침 없이 최신 상태만 다시 불러옵니다. 폰에서 불이 안 바뀌는 것 같을 때 눌러보세요.

**학생이 폰으로 풀 때 데이터가 아예 안 들어오는 경우:**  
- 학생 앱이 **폰**에서 문제를 풀었는데 `answer_logs` / 빨간불·파란불에 전혀 반영이 안 되면, **학생 앱 코드**에서 다음이 적용돼 있는지 확인하세요.
