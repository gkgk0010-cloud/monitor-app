# 오늘의 스코어 · 오늘의 약점 연동 — 지금 체크리스트

상세는 `1번연동_따라하기.md` 참고. 여기서는 **지금 할 일만** 정리합니다.

---

## 1. Supabase에 `answer_logs` 테이블 만들기 (한 번만)

1. **Supabase** 대시보드 → 사용 중인 **프로젝트** 선택
2. 왼쪽 **SQL Editor** → **New query**
3. **`monitor-app/supabase_answer_logs.sql`** 파일 내용 **전체 복사** → SQL Editor에 붙여넣기 → **Run**
4. **Success** 나오면 완료

→ 이걸 안 하면 모니터에서 "오늘 푼 기록이 없어요"만 나옵니다. (개인 로그는 `status_logs`라서 따로 보일 수 있음)

---

## 2. 똑패스 학생 앱 배포

1. **gas_src** 폴더에서 **`clasp push`** 실행
2. 똑패스 웹앱에서 **한 문제라도 풀기** (해당 학생 계정으로)
3. Supabase **Table Editor** → **answer_logs** 테이블 열어서 **방금 푼 문제 행이 생겼는지** 확인

→ GAS Script Properties에 **SUPABASE_URL**, **SUPABASE_ANON_KEY** 가 이미 있으면 (개인 로그가 보일 때와 동일 설정) 추가 설정 없음.

---

## 3. 모니터에서 확인

1. **모니터 앱** 새로고침 (또는 이미 배포돼 있으면 그대로)
2. **해당 학생 카드** 클릭 → 상세 팝업
3. **오늘의 스코어**에 "오늘 N문제 풀었고, 정답률 N%"  
   **오늘의 약점 (Worst 3)**에 태그별 오답 개수 (또는 "오늘 오답이 없어요") 가 나오는지 확인

- **오늘 한 번도 안 푼 학생**이면 "오늘 푼 기록이 없어요" → 정상
- **1단계 SQL을 아직 안 돌렸다면** → 1단계 실행 후 다시 확인

---

## 한 줄 요약

**Supabase에서 `supabase_answer_logs.sql` 실행 → `clasp push` → 학생이 문제 한두 개 풀기 → 모니터에서 해당 학생 카드 클릭해 보기.**
