# 똑패스 ↔ 선생님 모니터 연결 가이드

## 1. 전체 구조 (한눈에)

```
[학생] 똑패스 앱 (GAS 배포 index.html)
         │
         │ 로그인 → 퀴즈 풀기 → 정답/오답/복습완료/50문제 등
         │
         ▼ pushMonitorStatus() → Supabase API
         │
[Supabase] student_status 테이블
         │  (student_id, student_name, student_color, last_active, info_text)
         │
         ▼ 조회 / Realtime 구독
         │
[선생님] 모니터 앱 (monitor-app, Next.js)
         │  → 학생 명단, 상태 카드, 공지 복사 버튼 등
```

- **학생 명단·상태**는 똑패스 앱에서 **Supabase로 보내진 데이터**가 그대로 선생님 모니터에 나옵니다.
- 똑패스와 모니터는 **같은 Supabase 프로젝트·같은 테이블**만 맞추면 됩니다.

---

## 2. 똑패스 앱(In-App) 쪽 연결

### 2-1. 똑패스에서 하는 일

- **index.html** (학생이 쓰는 화면):
  - 로그인 성공 시 → `initStudentMonitorAfterLogin()` 호출
  - GAS에서 `getMonitorConfig()` 로 Supabase URL·Anon Key 받음 → 브라우저에서 Supabase 클라이언트 생성
  - 이후 학생 행동에 따라 `pushMonitorStatus(색, 설명)` 호출:
    - 접속 시: `white`, "접속 중"
    - 50문제 달성: `gold`, "🏆 일일 할당량(50문제) 클리어!"
    - 20문제 연속: `blue`, "🔥 열공 모드 (20문제+)"
    - 복습 완료: `green`, "✅ 복습 완료"
    - 3연속 오답: `red`, "3연속 오답"
    - 새벽 1~5시 접속: `purple`, "🟣 새벽반"
    - 샷건 종료: `white`, "🚪 샷건 종료 ..."
  - 보낼 때마다 **student_id**(로그인 uid), **student_name**, **student_color**, **info_text**, **last_active** 를 Supabase `student_status` 테이블에 **upsert** (같은 student_id면 갱신).

### 2-2. GAS (Code.js) 설정

- **Script Properties**에 다음 두 개 필수:
  - `SUPABASE_URL`: Supabase 대시보드 → Project Settings → API → Project URL
  - `SUPABASE_ANON_KEY`: 같은 화면의 anon public key
- 똑패스 웹앱이 GAS로 배포된 상태에서, 학생이 **로그인만 해도** 모니터용 Supabase 연결이 초기화되고, 위 상태들이 자동으로 테이블에 쌓입니다.

→ 이렇게 해 두면 **in-app 기능(상태 갱신, 50문제·복습·오답 등)** 이 모두 Supabase `student_status`와 연결됩니다.

---

## 3. Supabase 테이블 (사진 속 학생 명단)

### 3-1. 테이블 이름

- **`student_status`** 한 개만 사용합니다.

### 3-2. 필요한 컬럼

| 컬럼          | 타입        | 설명 |
|---------------|-------------|------|
| `id`          | int8 / uuid | PK (자동 생성 가능) |
| **`student_id`** | **text, unique** | **똑패스 로그인 uid (예: 김철수1234). upsert 기준.** |
| `student_name` | text       | 학생 이름 (멘트·생존자 명단에 사용) |
| `student_color`| text       | gold / red / blue / green / purple / white 등 |
| `last_active`  | timestamptz| 마지막 접속 시각 (생존자·미접속 카운트에 사용) |
| `info_text`    | text       | 상태 설명 (예: "3연속 오답", "🟣 새벽반") |

- **중요:** 똑패스 앱은 **`student_id`** 로 upsert합니다.  
  테이블에 `student_id` 컬럼이 없으면 **추가**하고, **unique 제약**을 걸어야 합니다.  
  (사진에는 `id`, `student_name`, `student_color`, `last_active`, `info_text`만 보일 수 있는데, `student_id`가 있어야 앱에서 보낸 데이터가 같은 학생으로 계속 갱신됩니다.)

### 3-3. Realtime (선택)

- 선생님 모니터에서 **실시간**으로 명단이 바뀌게 하려면:  
  Supabase 대시보드 → **Database → Replication** → `student_status` 테이블이 publication에 포함되도록 설정합니다.

---

## 4. 선생님 모니터(monitor-app) 쪽

- **monitor-app**은 Supabase **같은 프로젝트**의 `student_status`만 읽습니다.
- `.env.local`에 **NEXT_PUBLIC_SUPABASE_URL**, **NEXT_PUBLIC_SUPABASE_ANON_KEY** 를 똑패스에 넣은 것과 **동일한 값**으로 맞춥니다.
- 그러면:
  - **학생 명단** = `student_status` 테이블에 똑패스가 쌓아 둔 행들
  - **오늘의 생존자 / 미접속 카운트** = 같은 테이블의 `last_active`로 계산
  - **멘트 복사 / 공지 복사** = 같은 테이블의 `student_name`, `student_color` 사용

---

## 5. 정리: “in-app 기능”과 “사진 속 학생 명단” 연결

| 구분 | 연결 방법 |
|------|-----------|
| **In-app 기능** | 똑패스 index.html에서 `pushMonitorStatus()` → Supabase `student_status` **upsert** (GAS Script Properties에 URL·Anon Key 설정) |
| **학생 명단** | Supabase `student_status` 테이블을 선생님 모니터가 **조회** → 테이블에 쌓인 행 = 똑패스가 보낸 학생들 |
| **한 번만 맞추면 되는 것** | ① Supabase 테이블에 `student_id`(unique) 포함 ② 똑패스 GAS에 SUPABASE_URL, SUPABASE_ANON_KEY ③ 모니터 .env.local에 같은 URL·Anon Key |

이렇게 맞춰 두면 **똑패스만든 것(in-app 기능)** 과 **사진 속 학생 명단( Supabase 테이블)** 이 하나로 연결됩니다.
