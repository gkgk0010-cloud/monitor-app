# 2월 개강 전 전체 점검 — 분석 및 설명 (코드 수정 없음)

똑패스, 숙제인증, 결석영상인증, CCTV 모니터링이 **의도한 로직·공식·멘트**대로 동작하는지, **1월부터 유지해 온 설정**이 2월에도 그대로 적용되는지 **분석·설명**만 정리했습니다.

---

## 1. 월별 기준 (1월→2월 전환)

- **모든 월 기준**은 **한국 시간(GMT+9 / Asia/Seoul)** 으로 통일되어 있습니다.
- **시트/월 식별**은 `yyyy-MM` (예: 2026-01, 2026-02) 형태로 일관되게 사용됩니다.
- **현재 월**은 `getProfileMonthString(0)` 또는 `Utilities.formatDate(nowObj, "Asia/Seoul", "yyyy-MM")` 로 계산되므로, 2월 1일 0시(한국)부터 자동으로 2026-02가 적용됩니다.

---

## 2. 똑패스 (Code.js + GAS)

### 2-1. User_Profile (월별 시트)

| 항목 | 로직 | 2월 유지 여부 |
|------|------|----------------|
| 시트 이름 | `User_Profile_yyyy-MM` (예: User_Profile_2026-02) | ✅ 2월 들어가면 자동으로 2026-02 사용 |
| 새 시트 생성 | 1) `User_Profile`(무월) 2) 없으면 **지난달** `User_Profile_yyyy-MM` 을 템플릿으로 **1행(헤더) 전체 복사** | ✅ 1월 첫 행(태그 열 포함)이 2월에도 그대로 복사됨 |
| 기존 시트 보정 | 이미 있는 월별 시트가 **6열 이하**이면, 템플릿(지난달) 1행으로 **헤더 덮어쓰기** | ✅ 과거에 최소 헤더로 만들어진 2월 시트도 접근 시 자동 보정 |

- **정리:** 1월에 쓰던 User_Profile 구조(첫 행·열 수)가 2월 신규 생성/보정 시 그대로 이어집니다.

### 2-2. 오답노트 (월별 + 월마다 초기화)

| 항목 | 로직 | 2월 유지 여부 |
|------|------|----------------|
| 시트 이름 | `오답노트_yyyy-MM` (예: 오답노트_2026-02) | ✅ 2월부터 오답노트_2026-02 사용 |
| 데이터 | **월이 바뀌면 새 시트** → 헤더만 있고 데이터는 비움 (월마다 초기화) | ✅ 1월 오답은 1월 시트에만 있고, 2월은 새로 쌓임 |

- **정리:** 1월 오답 데이터는 1월 시트에 보존되고, 2월은 빈 시트로 시작하는 설계가 유지됩니다.

### 2-3. 로그 시트 (Log_YYYY-MM_A/B)

- 시트 이름: `Log_yyyy-MM_A` (1~15일), `Log_yyyy-MM_B` (16일~말일).
- 2월이 되면 `Log_2026-02_A` / `Log_2026-02_B` 가 필요 시 생성되며, 기존 로직 그대로입니다.

## 3. 숙제인증 (Homework.js)

### 3-1. 월별 시트 및 첫 행(헤더)

| 항목 | 로직 | 2월 유지 여부 |
|------|------|----------------|
| 시트 이름 | `yyyy-MM` (예: 2026-02) | ✅ 현재 월이 2월이면 2026-02 시트 사용 |
| 새 시트 생성 | 지난달 시트가 있으면 **1행 중 앞 6열만** 복사 (ID, 이름, 생년월일, 가입일, 점수, 방이름). **일차 열(7열~)은 복사하지 않음** | ✅ 2월은 기본 6열로 시작하고, 일차는 **인증할 때마다** 추가되는 설계 유지 |
| 일차 열 | 헤더에 해당 일차가 없으면 **auth(제출) 시점에** 1열 추가 | ✅ “처음부터 1일차~17일차 다 열려 있으면 안 된다”는 요구사항대로 동작 |

### 3-2. register (가입 / 로그인 시 반 반영)

| 항목 | 로직 | 멘트 예시 |
|------|------|-----------|
| 신규 | A열에 없으면 한 행 추가 (ID, 이름, 생년월일, 가입일, 점수, 방이름) | `"OO님(생년월일) 등록 완료! [방이름] 배정됨. 😎"` |
| 기존(앱 회원가입 후 첫 로그인) | **이미 있으면** 3열(생년월일), 6열(방이름)만 갱신 | `"✅ 반 정보가 반영되었습니다. [방이름]"` |

- **정리:** 로그인 시 선택한 반이 숙제 시트 “방이름”에 반영되는 로직이 유지됩니다.

### 3-3. auth (제출) — 진도·지각·점수

| 항목 | 로직 | 멘트 예시 |
|------|------|-----------|
| 진도 차단 | `dayNum > currentClassMaxDay + 1` 이면 제출 불가 | `"🚫 [N일차] 아직 진도가 안 나갔어요! 현재 (currentClassMaxDay+1)일차까지만 가능합니다."` |
| 중복 제출 | 해당 일차 셀에 이미 값이 있으면 불가 | `"이미 N일차 인증 완료 (O/지각/링크)"` |
| 지각 여부 | `checkHomeworkStatus`: 오늘 방 열림이면 정시, 그 외 조건(요일·시간·휴강)에 따라 지각 | — |
| 점수 | 정시 1점, 지각 0.5점 누적 | 정시: `"[🎉 N일차 과제 인증 성공!] ⏰ HH:mm 💯 1점 (누적 N점) 📸 카페 인증 필수!"` / 지각: `"[⚠️ N일차 지각 제출] ⏰ HH:mm 📉 0.5점 (누적 N점) 📸 카페 인증 필수!"` |

- **정리:** 1월에 쓰던 진도·지각·점수·멘트 규칙이 2월 시트에서도 동일하게 적용됩니다.

### 3-4. 휴강 / 리포트 (메신저봇)

- 휴강: `processHomeworkHoliday` → LAST_HOLIDAY 또는 HOLIDAY_방이름 저장. 멘트에 “오늘 오후 및 내일 오전 제출까지 지각 면제” 안내.
- 리포트: `processHomeworkReportOnly` → “📅 MM-dd 숙제 현황 (방이름 - N일차) ⭕ 제출(N명) ❌ 미제출(N명)” 형식 유지.

## 4. 결석영상인증 (Absence.js)

### 4-1. 학기(월) 목록 — TARGET_MONTHS

| 항목 | 현재 값 | 2월 개강 시 |
|------|---------|-------------|
| TARGET_MONTHS | `["2026-03", "2026-02", "2026-01", "2025-12"]` | ✅ **2026-02 포함** → 2월 개강 학기로 인식됨 |

- **운영 참고:** 이후 2026-04 등 새 학기를 쓰려면 이 배열에 `"2026-04"` 를 추가해야 합니다. 코드 수정 필요.

### 4-2. 월별 시트 및 첫 행 복사

| 시트 종류 | 이름 예시 | 새 시트 생성 시 |
|-----------|-----------|-----------------|
| 학생기록 | `yyyy-MM_학생기록` (예: 2026-02_학생기록) | 지난달 `yyyy-MM_학생기록` **1행 전체** 복사. 없으면 기본 8열 헤더 |
| 영상DB | `yyyy-MM_영상DB` (예: 2026-02_영상DB) | 지난달 `yyyy-MM_영상DB` **1행+데이터 행 전체** 복사. 없으면 일차 1~20 구조로 생성 |

- **정리:** 1월에 쓰던 “첫 행 구조”가 2월 신규 생성 시 그대로 이어집니다.

### 4-3. 주요 멘트

| 상황 | 멘트 |
|------|------|
| 반 등록 완료 | `"✅ [yyyy-MM학기] 등록 완료! 👤 이름: OO 🏫 반: OO"` |
| 미인증(빚) 안내 | `"📝 [N일차] 인증을 아직 안 하셨네요! 지금 인증하시겠습니까?"` |
| 빚 있는데 다른 일차 요청 | `"✋ [N일차] 인증이 아직 안 됐어요! 밀린 것부터 해결해주세요."` |
| 인증 정답 | `"🎉 정답입니다! 도장 쾅! 💮 (총 N개 완료)"` |
| 인증 오답 | `"땡! 😅 코드가 틀렸어요."` |

- **정리:** 1월과 동일한 멘트·플로우가 2월에도 유지됩니다.

## 5. 회원가입 시 외부 시트 (Code.js doRegister)

| 대상 | 시트 이름 | 새 시트 생성 시 | 추가 행 구조 |
|------|-----------|-----------------|--------------|
| 숙제 시트 | `yyyy-MM` (2026-02) | 지난달 1행 **앞 6열만** 복사. 없으면 6열 기본 헤더 | ID, 이름, "", 가입일, 0, "" (방이름은 로그인 시 숙제 register로 채움) |
| 결석 시트 | `yyyy-MM_학생기록` (2026-02_학생기록) | 지난달 **1행 전체** 복사. 없으면 8열 기본 헤더 | ID, 이름, "", "", 0, 0, "", new Date() |

- **정리:** 2월 첫 회원가입 시 2026-02(숙제), 2026-02_학생기록(결석)이 위 규칙으로 생성·추가되며, 1월과 동일한 “첫 행 유지” 정책이 적용됩니다.

## 6. CCTV 모니터링 (monitor-app)

### 6-1. 데이터 소스

| 항목 | 설명 |
|------|------|
| 학생 목록·상태 | Supabase `student_status` (Realtime 구독) |
| 사건 기록 | Supabase `status_logs` (INSERT 구독) |
| 오늘 스코어·약점 | Supabase `answer_logs` (학생별·오늘 한국날짜 필터) |

- **정리:** 1월에 연결해 두신 Supabase 프로젝트·테이블이 그대로 2월에도 사용됩니다. 월 전환 별도 설정 없음.

### 6-2. 집중관리존 (파란불/빨간불)

| 항목 | 로직 |
|------|------|
| 표시 기준 | `student_status.last_answer_result` + `last_answer_at` + `last_answer_tag` |
| “최근” 정의 | `last_answer_at` 이 **10초 이내**면 정답/오답 표시, 그 외는 “대기” |
| 정답 | 파란불 + “✅ 정답” (태그 있으면 “✅ 정답 · 태그명”) |
| 오답 | 빨간불 + “❌ 오답” (태그 있으면 “❌ 오답 · 태그명”) |
| 갱신 방식 | 학생 앱이 문제 제출 시 `pushAnswerResultToMonitor(isCorrect, tag)` 로 upsert → Realtime으로 모니터에 즉시 반영 (새로고침 불필요) |

- **정리:** 의도한 “문제 풀 때마다 정답=파란불, 오답=빨간불, 태그 표시” 로직이 유지됩니다. 주요 상황(3연속 오답, 50문제 클리어 등)은 실시간 사건 기록에만 쌓입니다.

### 6-3. 실시간 사건 기록

- `status_logs` INSERT 구독으로 GOLD, RED, BLUE, GREEN, PURPLE, SHOTGUN 등 이벤트가 실시간 표시됩니다.
- 1월부터 쓰던 이벤트 타입·멘트가 그대로 2월에도 기록·표시됩니다.

### 6-4. 오늘의 스코어·오늘의 약점

- `answer_logs` 에서 **오늘(한국 날짜)** 데이터만 필터해 정답률·Worst 3 태그 계산.
- 월이 2월로 바뀌어도 “오늘”만 보므로 별도 전환 로직 없이 2월 개강 후에도 동일하게 동작합니다.

| 구분 | 1월에 쓰던 것 | 2월 전환 시 |
|------|----------------|-------------|
| User_Profile | User_Profile_2026-01, 첫 행(태그 등) | User_Profile_2026-02 생성/보정 시 1월 첫 행 복사 또는 보정으로 **동일 구조 유지** |
| 숙제인증 시트 | 2026-01, 6열+인증 시 일차 열 추가 | 2026-02 동일 규칙, **첫 행 6열만 복사**·일차는 인증 시 추가 |
| 결석영상 시트 | 2026-01_학생기록, 2026-01_영상DB | 2026-02_* 생성 시 **1월 첫 행(또는 전체 구조) 복사** |
| 오답노트 | 오답노트_2026-01 | 오답노트_2026-02 **신규** (월마다 초기화) |
| 모니터 | student_status, status_logs, answer_logs | **동일 테이블**·Realtime·쿼리, 월 전환 없음 |
| 시간 기준 | 한국 시간(GMT+9 / Asia/Seoul) | **동일** |

## 8. 운영 시 확인하면 좋은 것 (코드 수정 없이 점검)

1. **결석영상 TARGET_MONTHS**  
   - 2026-04 학기부터 쓰려면 Absence.js의 `TARGET_MONTHS` 에 `"2026-04"` 추가가 필요합니다. 2월 개강 시점에는 2026-02가 이미 포함되어 있어 **추가 작업 없음**.
2. **Supabase Realtime**  
   - `student_status` 가 **Database → Publications → supabase_realtime** 에 포함되어 있는지만 확인하면 됩니다. (이미 켜 두셨다면 2월에도 동일.)
3. **GAS Script Properties**  
   - SUPABASE_URL, SUPABASE_ANON_KEY 등 1월에 설정한 값이 그대로라면 2월에도 동일하게 적용됩니다.

---

**정리:**  
코드 상으로는 **월 기준(yyyy-MM)·한국 시간·첫 행 복사·일차 추가 타이밍·멘트·CCTV/사건기록/스코어 로직**이 모두 1월 설계를 유지한 채 2월에 자동으로 적용되도록 되어 있습니다.  
위 항목만 실제 환경에서 한 번씩만 확인해 보시면, “의도한 로직·공식·멘트가 전부 정상이고, 1월부터 진행해 오던 것이 잘 유지되고 있다”고 점검하실 수 있습니다.
