# 배포 딱 3단계 (초간단)

지금 보이는 Vercel 화면에서 **이 순서대로만** 하면 됩니다.

---

## ① GitHub에 코드 올리기 (한 번만)

1. **GitHub** (github.com) 로그인
2. **새 저장소** 만들기: 초록색 **New** 버튼 → Repository name에 `monitor-app` 입력 → **Create repository**
3. **monitor-app 폴더만** 그 저장소에 올리기:
   - 방법 A: GitHub 웹에서 **uploading files** 로 `monitor-app` 안에 있는 파일들 전부 끌어다 놓기 (폴더 안에 app, utils, package.json 등이 보이게)
   - 방법 B: PC에서 Git 쓰는 경우 — monitor-app 폴더로 들어가서 터미널에 아래 입력 (저장소 주소는 본인 걸로 바꾸기)
     ```
     git init
     git add .
     git commit -m "first"
     git branch -M main
     git remote add origin https://github.com/본인아이디/monitor-app.git
     git push -u origin main
     ```

---

## ② Vercel에서 "Import" 하기

1. 지금 보이는 Vercel 화면에서 **"Import Project"** 카드 찾기
2. **Import** 버튼 클릭
3. **GitHub** 선택 → 권한 허용
4. 나오는 목록에서 **방금 만든 저장소** (예: monitor-app) 선택
5. **Import** 클릭

---

## ③ 환경 변수 넣고 배포

1. **Configure Project** 화면이 나오면:
   - **Root Directory** 옆 **Edit** 클릭 → `monitor-app` 입력 (저장소 이름이 이미 monitor-app이면 비워도 됨)
   - 아래로 내려서 **Environment Variables** 펼치기
2. **Add** 해서 아래 두 개 입력:

   | 이름 (Key) | 값 (Value) |
   |------------|------------|
   | `NEXT_PUBLIC_SUPABASE_URL` | Supabase 대시보드 → Project Settings → API → **Project URL** 복사해서 붙여넣기 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 같은 화면에서 **anon public** 키 복사해서 붙여넣기 |

3. **Deploy** 버튼 클릭
4. 끝나면 **Visit** 누르거나, 주소창에 `https://만들어진주소.vercel.app/teacher/monitor` 로 들어가기

---

## 막히는 곳이 있으면

- **GitHub에 뭘 올려야 해?** → monitor-app **폴더 안에 있는 것 전부** (app, utils, package.json, next.config.js 등). `.next` 폴더는 안 올려도 됨.
- **Supabase 주소·키 어디서 찾아?** → supabase.com 로그인 → 프로젝트 선택 → 왼쪽 아래 **Project Settings** → **API** → Project URL / anon public 복사.
- **Root Directory가 뭐야?** → 저장소에 monitor-app **폴더만** 올렸으면 비워 두고, 똑패스 전체를 올렸으면 `monitor-app` 입력.

이 3단계만 하면 배포 끝입니다.
