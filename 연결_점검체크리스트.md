# Supabase 연결 안 될 때 점검

## 1. ERR_NAME_NOT_RESOLVED 가 나오는 경우

- **의미:** 브라우저가 `chfmyqdbnvrudumckvat.supabase.co` 주소를 찾지 못함 (DNS 실패).

### 체크할 것

1. **URL 오타**
   - `.env.local` 에 **chfmyq** (q) 인지 확인. **chfmyg** (g) 로 되어 있으면 수정.
   - Supabase 대시보드 → **Project Settings** → **API** → **Project URL** 복사해서 그대로 붙여넣기.

2. **캐시 제거 후 재실행**
   ```bash
   cd monitor-app
   rmdir /s /q .next
   npm run dev
   ```
   브라우저에서 **Ctrl+Shift+R** 강력 새로고침.

3. **Supabase 점검/유지보수**
   - 대시보드 상단에 "Scheduled maintenance" 가 떠 있으면 점검 끝날 때까지 대기.

4. **네트워크**
   - 회사/학교 네트워크·방화벽에서 `*.supabase.co` 차단 여부 확인.
   - 휴대폰 핫스팟으로 바꿔서 같은 페이지 접속해 보기.

5. **프로젝트 일시정지**
   - 무료 플랜은 비활성 기간 뒤 프로젝트가 일시정지됨. 대시보드에서 "Resume project" 로 재개.

---

## 2. 연결 실패 시 화면 안내

- 모니터 페이지에서 Supabase 요청이 실패하면 **"Supabase 연결 실패"** 박스와 위 체크 항목이 화면에 표시됨.
