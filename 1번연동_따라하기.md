# 1번 연동 따라하기 (오늘의 스코어 · 오늘의 약점)

아래 **1 → 2 → 3** 순서대로 하면 됩니다.

---

## 1단계: Supabase에 테이블 만들기 (한 번만)

1. **Supabase** 대시보드 접속 → 사용 중인 프로젝트 선택
2. 왼쪽 메뉴에서 **SQL Editor** 클릭
3. **New query** 클릭
4. 프로젝트 안의 **`monitor-app/supabase_answer_logs.sql`** 파일을 열어서 **전체 내용 복사**
5. SQL Editor 창에 **붙여넣기** 후 **Run** (또는 Ctrl+Enter) 클릭
6. "Success" 나오면 1단계 끝.  
   (에러가 나오면 메시지 그대로 복사해서 알려주세요.)

→ 이렇게 하면 **answer_logs** 테이블이 생깁니다.  
학생이 문제 풀 때마다 "누가, 언제, 어떤 태그, 정답/오답" 한 줄씩 여기에 저장됩니다.

---

## 2단계: 똑패스 앱에 코드 반영하기

똑패스 **학생 앱** 쪽에는 이미 아래가 들어가 있습니다.

- 문제 제출 후 **저장이 성공하면** → 같은 Supabase의 **answer_logs**에 한 줄 넣는 함수 **pushAnswerLog**
- **일반 모드** · **복습 모드** 모두에서 `saveResult` 성공 시 `pushAnswerLog(uid, 이름, 태그, 정답여부)` 호출

**할 일:**

1. **gas_src** 수정본이 있다면 **clasp push** 로 GAS에 배포
2. 똑패스 웹앱에서 **한 문제라도 풀어보기**  
   → Supabase **Table Editor**에서 **answer_logs** 테이블을 열어 보면 방금 푼 문제가 한 줄 들어와 있어야 합니다.

여기까지 되면 "문제 풀 때마다 Supabase에 기록하기" 연동은 끝입니다.

---

## 3단계: 모니터 앱에서 확인하기

모니터 앱에는 이미 아래가 들어가 있습니다.

- **학생 카드 클릭 → 상세 팝업**이 뜰 때  
  해당 학생의 **student_id**로 **answer_logs**를 조회해서  
  **오늘(한국시간)** 데이터만 모아서:
  - **오늘의 스코어:** 푼 문제 수, 정답 수, 오답 수, 정답률
  - **오늘의 약점 (Worst 3):** 오답만 모아서 태그별 개수 → 상위 3개

**할 일:**

1. **monitor-app** 수정본을 **빌드/배포** (로컬면 `npm run dev`, 배포면 GitHub push 후 Vercel 등)
2. 모니터 페이지에서 **학생 카드 클릭** → 상세 팝업 열기
3. **📊 오늘의 스코어**에 "오늘 N문제 풀었고, 정답률 N%"  
   **📉 오늘의 약점 (Worst 3)**에 "가정법 3개, 관계사 1개 …" 같은 문구가 나오는지 확인

- 오늘 한 번도 안 푼 학생이면 "오늘 푼 기록이 없어요"로 나오는 것이 정상입니다.
- 1단계에서 **answer_logs** 테이블을 아직 안 만들었다면, 팝업에는 "오늘 푼 기록이 없어요" 또는 에러 처리로 비슷한 문구가 나올 수 있습니다.  
  → 1단계 SQL 실행 후 다시 확인하면 됩니다.

---

## 요약 체크리스트

- [ ] **1단계** Supabase SQL Editor에서 `supabase_answer_logs.sql` 실행 → answer_logs 테이블 생성
- [ ] **2단계** gas_src에 pushAnswerLog 반영 후 clasp push → 똑패스에서 문제 한 번 풀기 → answer_logs에 행 생기는지 확인
- [ ] **3단계** monitor-app 배포 후 학생 카드 클릭 → 상세 팝업에서 오늘 스코어·약점 문구 확인

이 순서대로 하시면 1번 연동을 바로 따라 하실 수 있습니다.
