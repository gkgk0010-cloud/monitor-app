# Supabase student_id 형식 안 맞는 학생 정리 방법

**맞는 형식**: `이름+휴대폰 뒤 4자리` (예: 노수진1234, 허진혁8657)  
**안 맞는 예**: `이름만` (노수진, 제갈수용), `u+숫자` (u1770013815621) 등

---

## 1. 누가 안 맞는지 찾기

**Supabase 대시보드** → Table Editor → `student_status` 테이블에서:

- **student_id**가 **숫자 4자리로 끝나지 않는** 행 = 형식 안 맞음  
  (예: "노수진", "제갈수용", "u1770013815621")
- 또는 **SQL Editor**에서 아래 쿼리로 목록 뽑기:

```sql
-- student_id가 "이름+숫자4자리"가 아닌 행 (끝 4자가 숫자가 아니거나, 길이가 5 미만)
SELECT id, student_id, student_name, last_active
FROM student_status
WHERE student_id !~ '.\d{4}$'   -- 끝 4자리가 숫자로 끝나지 않음
   OR length(trim(student_id)) < 5
ORDER BY last_active DESC NULLS LAST;
```

---

## 2. 어떻게 맞추면 되는지 (선택지)

### 방법 A: 학생에게 다시 로그인 요청 (추천)

1. 위에서 **형식 안 맞는 목록** 확인 (이름만 / u+숫자 등).
2. 해당 학생들에게 **단톡/공지**로 안내:
   - *"앱 로그인할 때 **이름 + 휴대폰 뒤 4자리** 로 입력해 주세요. 예: 노수진1234"*
3. 학생이 **이름+4자리**로 다시 로그인한 뒤 문제라도 한 번 풀면,  
   Supabase에 **student_id = "노수진1234"** 로 새로 들어가거나(또는 upsert로 갱신됨).
4. **예전 행**(student_id가 "노수진"인 행)은:
   - 그냥 두어도 되고,
   - 중복이 싫으면 Supabase Table Editor에서 **해당 행만 삭제**해도 됨.

→ **장점**: 앞으로 그 학생은 항상 "노수진1234" 형식으로만 쌓임.  
→ **단점**: 학생이 다시 로그인할 때까지는 예전 행이 남아 있음.

---

### 방법 B: Supabase에서 수동으로 student_id 수정

1. 학생에게 **휴대폰 뒤 4자리**를 물어봐서 확인.
2. Supabase **Table Editor** → `student_status` → 해당 행 클릭  
   → **student_id** 값을 `이름+4자리`로 수정 (예: `노수진` → `노수진1234`).
3. 저장.

→ **장점**: 학생이 다시 로그인 안 해도 테이블만 바로 통일 가능.  
→ **단점**: 나중에 그 학생이 앱에서 "노수진1234"로 로그인하면, **같은 student_id**로 upsert되므로 그 행이 갱신됨. (수정한 값이 "노수진1234"면 문제없음.)

**주의**: student_id를 잘못 바꾸면(다른 사람 4자리로) 그 학생이 로그인했을 때 다른 행이 갱신될 수 있으니, 꼭 본인 4자리로 수정할 것.

---

### 방법 C: u+숫자 행 정리

- **u1770013815621** 같은 건, 예전에 앱에서 `currentUser.id`를 썼을 때 생긴 값일 수 있음.
- 해당 학생이 **이름+4자리**로 다시 로그인해서 활동하면, **student_id = "이름1234"** 인 행이 생김.
- 그러면 **u+숫자 행**은:
  - **student_name**으로 같은 사람인지 확인한 뒤,
  - 같은 사람이면 **u+숫자 행 삭제**하고, **이름+4자리** 행만 두면 됨.

---

## 3. 요약

| 상황 | 추천 |
|------|------|
| **이름만** (노수진, 제갈수용) | 학생에게 "이름+4자리로 다시 로그인" 안내 → 새로 쌓이는 행만 쓰거나, 예전 행 삭제 |
| **u+숫자** | 같은 학생이 "이름+4자리"로 로그인 후 생긴 행만 두고, u+숫자 행 삭제 |
| **앞으로** | 로그인 시 이미 "이름+4자리"만 허용하므로, 새로 들어오는 건 전부 형식 통일됨 |

정리하면: **안 맞는 애들은**  
1) **누가 안 맞는지** SQL/테이블로 찾고,  
2) **학생에게 다시 로그인(이름+4자리)** 요청하거나,  
3) **Supabase에서 student_id만 수동 수정**해서 맞추면 됨.
